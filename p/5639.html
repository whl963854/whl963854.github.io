    类型推断

    模板型别推导

    模板的型别推导是现代c++最广泛应用的特性之一————————auto的基础，了解作为auto基础的模板型别推导就十分重要。
    函数模板大致形如：

    template<typename T>
void f(ParamType param);

    而一次调用则形如：

    f(expr);   //以某个表达式调用f

    在编译期，编译器会通过expr推导两个型别：一个是T的型别，另一个是ParamType 的型别，两个型别往往不一样，因为ParamType常会包含了一些修饰词，如const或引用符号等限定词。例如，若模板声明如下：

    template<typename T>
void f(const T& param);

    而调用语句如下：

    int x=0;
f(x);

    在此例中，T会被推导为int，ParamType则被推导为const int&。
    但具体而言，T的型别推导结果不仅依赖于expr的型别，还依赖于ParamType 的形式，具体分为三种情况讨论：

    • ParamType具有指针或者引用型别，但不是万能引用
    • ParamType是一个万能引用
    • ParamType既非指针也非引用

    ParamType具有指针或者引用型别，但不是万能引用

    1. 若expr具有引用型别，先将引用部分忽略
    2. 对expr的型别和ParamType 的型别执行模式匹配，来决定T的型别。

    例如模式如下：

    template<typename T>
void f(T& param);

    又声明了下列变量：

    int x = 27;
const int cx = x;
const int& rx = x;

    在各次调用中，对param和T 的型别推导结果如下：

    f(x);   //T的型别是int，param的型别是int&
f(cx);  //T的型别是const int，param的型别是const int&
f(rx);  //T的型别是const int，param的型别是const int&

    rx 的引用性会在型别推导过程中被忽略

    如果我们在ParamType中加入const属性：

    template<typename T>
void f(const T& param);

int x = 27;
const int cx = x;
const int& rx = x;

f(x);   //T的型别是int，param的型别是const int&
f(cx);  //T的型别是int，param的型别是const int&
f(rx);  //T的型别是int，param的型别是const int&

    同上，rx 的引用性会在型别推导过程中被忽略。

    如果param是一个指针，而非引用，运作方式本质上并无不同：
    template
    void f(T* param);

    int x = 27;
const int *px = &x;

f(&x);   //T的型别是int，param的型别是int*
f(px);  //T的型别是const int，param的型别是const int*

    ParamType是一个万能引用

    1. 如果expr是个左值，T和ParamType都会被推导为左值引用。
    2. 如果expr是个右值，则应用第一条规则。

    例如：

    template<typename T>
void f(T&& param);

int x = 27;
const int cx = x;
const int& rx = x;

f(x);   //T的型别是int&，param的型别是int&
f(cx);  //T的型别是const int&，param的型别是const int&
f(rx);  //T的型别是const int&，param的型别是const int&
f(27);  //T的型别是int，param的型别是int&&

    ParamType既非指针也非引用

    1. 若expr具有引用型别，先将引用部分忽略
    2. 若expr是个const,volatile对象，则忽略之

    例如：

    template<typename T>
void f(T param);

int x = 27;
const int cx = x;
const int& rx = x;

f(x);   //T的型别是int，param的型别是int
f(cx);  //T的型别是int，param的型别是int
f(rx);  //T的型别是int，param的型别是int

    param是个完全独立于cx和rx的对象，是他的一个副本，即使expr不可修改，param也和他无关。
    需要说明的是，expr是个指向const对象的const指针，且expr按值传给param：

    template<typename T>
void f(T param);

const char* const ptr = "test";

f(ptr);   //T的型别是const char*，param的型别是const char*

    数组实参

    数组型别有别于指针型别，数组会退化成首元素的指针。

    const char name[] = "Test";    //name的型别是const char[5]; '\0'
const char* ptrToName = name;  //数组退化成指针 const char *

    当一个数组传递到持有按值形参的模板时

    template<typename T>
void f(T param);
f(name);

void myFunc(int param[]);
//这意味着myFunc等价的声明如下：
void myFunc(int* param);

    这种数组和指针形参的等价性，是c++基础的c根源遗迹，它使得“数组和指针型别是一回事”
    由于数组形参声明会按照他们好像是指针形参那样加以处理，按值传递给函数模板的数组型别将被推导成指针型别。
    也就是说，在模板f的调用中，其类型形参T会被推到成const char*

    f(name);   //name是个数组，但是却被推导成了const char*

    尽管函数无法声明数组类型的形参，他们却可以声明形参为数组的引用！

    template<typename T>
void f(T& param);
f(name);

    在这种情况下，T会被推导成实际的数组类型，这个型别会包含数组的尺寸 T 的推导类型为const char[5];param则被推导为const char(&)[5];

    template<typename T,std::size_t N>
constexpr std::size_t arraySize(T (&)[N])noexception
{
    return N;
}

    我们可以利用模板推导出数组含有的元素个数。

    函数实参

    和数组一样，函数也会退化成函数指针

    void someFunc(int,double);
template<typename T>
void f1(T param);

template<typename T>
void f2(T& param);

f1(someFunc);  //param具体型别 void (*)(int,double)
f2(someFunc);  //param具体型别 void (&)(int,double)

    auto 型别推导

    auto推导规则和模板类型推导大致相同，也是分为三种类型

    • ParamType具有指针或者引用型别，但不是万能引用
    • ParamType是一个万能引用
    • ParamType既非指针也非引用

    这里不进行细致研究，下面是一些例子，规则参考模板类型推导：

    auto x = 27;
const auto cx = x;
const auto& rx = x;
auto&& uref1 = x;// int&
auto&& uref1 = cx;// const int&
auto&& uref1 = 27;// int&&

    同时，关于数组指针以及函数指针相关退化规则也完全相同：

    const char name[] = "test";
auto arr1 = name;// const char*
auto arr2 = name;// const char (&)[5]
void someFunc(int,double);
auto func1 = someFunc;// void (*)(int,double)
auto& func2 = someFunc;// void (&)(int,double)

    但是auto类型推导有一个例外，由于c++支持统一初始化

    int x3 = {2};
auto x4 = {2};

    这里x3是int类型，但是x4类型并不是int，而是std::initializer_list
    这也是auto和模板类型推导的主要区别，例如：

    auto x = {1,2,3,4,5};// x的类型是std::initializer_list<int> 

template<typename T>
void f(T param);

f({1,2,3,4,5});//错误，无法推导T的型别

    但是如果你指定模板参数类型为std::initializer_list 推导机制则会推导出T的类型

    template<typename T>
void f(std::initializer_list<T> param);

f({1,2,3,4,5});//推导T的型别为int

    auto 作为函数返回类型

    c++处理函数时使用类型推断的情况有两种类型

    • 创建一个模板函数并在没有显式具体化的情况下调用该函数，可以推断函数参数的类型
    • 代替函数的返回值类型

    例如：

    auto AutoFunction(int t)
{
    return t;
}

    c++11 这里会出现错误，后续版本解决了这个问题。
    c++11解决该问题的方法是指定函数返回类型：

    auto AutoFunction(int t)->int
{
    return t;
}

    但是如果在模板函数中，返回类型不确定，这时候需要引入一个新的关键字来进行推导：decltype

    template<typename T>
auto AutoFunction(T t)->decltype(t)
{
    return t;
}

    c++14 纠正了这一情况，即使是和模板一起使用auto推导返回值类型也不需要后置返回值类型。
    在C++14中，编译器可以直接推断出函数的返回类型了，所以可以写成下边的样子：

    decltype(auto) AutoFunction(int t)->int
{
    return t;
}  

//或者直接下面这种方式也可以 
auto AutoFunction(int t)->int
{
    return t;
}

    但是二者有所区别，auto会自动去除返回值的引用性，但是decltyoe会保留原有属性，
    在一般情况下，decltype都是返回给定表达式或者给定的名字的类型，但是在某些情况下还是有点问题,如果仅有一个名字，那么decltype的行为保持不变，但如果是一个复杂度左值表达式，decltype会返回一个左值引用。

    //返回int
delctype(auto) f1()
{
    int x = 0;
    ....
    return x;
}

//返回int&
delctype(auto) f1()
{
    int x = 0;
    ....
    return (x);
}

    auto的使用还需要注意一下代理类相关的问题。比如：

    std::vector<bool> features(const Widget& w);
Widget w;
auto temp = features(w)[5];

    temp的类型并不是我们想象中的bool，这是因为vector对bool类型做过特化，每一个bool元素用一个比特位进行表示，返回值并不是bool类型，而是std::vector::reference;std::vector::reference能够向bool进行隐式类型转换
